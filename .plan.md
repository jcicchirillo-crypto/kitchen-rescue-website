# Fix White Box Text Color and Footer Alignment

## Issues

1. **Text still faint in white boxes**: There's a CSS rule (lines 1266-1272) setting the container div to `color: #111827 !important` (light grey) which is overriding the paragraph text color
2. **Footer margins don't line up**: Need to verify the footer container padding matches the content boxes above
3. **White info boxes don't match bigger margin width**: The white boxes in the spec section need to match the width/padding of other sections

## Changes Required

### 1. Remove or update the container color rule (styles.css, lines 1266-1272)

**Current:**
```css
.spec [style*="background: white"],
.spec [style*="background:#f9fafb"],
.spec [style*="background: #f9fafb"],
.spec [style*="background:#ffffff"],
.spec [style*="background:white"] {
    color: #111827 !important;
}
```

**Solution: Delete this entire rule** - We already have specific rules for h4 (red) and p (black), so the container doesn't need a color override.

### 2. Verify footer padding matches content sections

Check that `.footer .container` has `padding: 0 36px` to match the 36px horizontal padding of all other sections.

### 3. Check white box alignment in spec section

The white info boxes should be inside the spec section's grid and inherit proper spacing automatically from their parent.
